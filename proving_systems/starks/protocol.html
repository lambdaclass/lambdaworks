STARKs protocol

In this section we describe precisely the STARKs protocol used in Lambdaworks.

Definitions and notation

  • Program size: 2n rows (number of states) and m′ original columns (number of registers) and m′′ RAP extended columns. Let m=m′+m′′.
  • Boundary constraints polynomials PjB​ for j=1,…,m.
  • Boundary constraint zerofiers ZjB​ for j=1,…,m..
  • Quadratic transition constraint polynomials PkT​ for k=1,…,nT​.
  • Transition constraint zerofiers ZjT​ for k=1,…,nT​.
  • b=2l the blowup factor
  • ω a primitive 2n+l-th root of unity.
  • g=ω2l.
  • The Small domain is the vector DS​=(1,g,…,g2n−1).
  • h the coset factor.
  • Q number of FRI queries.
  • Low Degree Extension is the vector DLDE​=(h,hω,hω2,…,hω2n+l−1).
  • Given a vector A=(y1​,…,yL​) and a function f:set(A)→F, denote by f(A) the vector (f(y1​),…,f(yL​)). Here set(A) denotes the underlying set of A.
  • D the smallest power of two larger than n⋅max{deg(PkT​),deg(PjB​)}k,j​
  • A polynomial p∈F[X] induces a function f:A→F for every subset A of F, where f(a):=p(a).
  • Two polynomials p,q∈F[X] induce a function f:A→F for every subset A disjoint from the set of roots of q, where f(a):=p(a)q(a)−1. Denote f by p/q.
  • A hash function from F to binary strings of fixed length.

Randomized AIR with Preprocessing (RAP)

This the process in which the prover uses randomness from the verifier to complete the program trace with additional columns. This is specific to each RAP. See here for more details.

Cairo's RAP

The execution of a Cairo program produces a memory vector V and a matrix M of size L×3 with the evolution of the three registers pc,ap,fp. Both with entries in F.

First round of RAP:
  1. Augment each row of M with information about the pointed instruction as follows: For each entry (pci​,api​,fpi​) of M, unpack the pci​-th value of V. The result is a new matrix M∈FL×33 with the following layout
 A.  flags   (16) : Decoded instruction flags
 B.  res     (1)  : Res value
 C.  mem_p   (2)  : Temporary memory pointers (ap and fp)
 D.  mem_a   (4)  : Memory addresses (pc, dst_addr, op0_addr, op1_addr)
 E.  mem_v   (4)  : Memory values (inst, dst, op0, op1)
 F.  offsets (3)  : (off_dst, off_op0, off_op1)
 G.  derived (3)  : (t0, t1, mul)

 A                B C  D    E    F   G
├xxxxxxxxxxxxxxxx|x|xx|xxxx|xxxx|xxx|xxx┤

  1. Let R be the last row of M, and let R′ be the vector that's equal to R except that it has zeroes in entries corresponding to the set of columns mem_a, mem_v. Let Lpub​ be the length of the public input (program code). Extend M with additional L′:=⌈Lpub​/4⌉ rows to obtain a matrix M∈F(L+L′)×33 by appending copies of the row R′ at the bottom (the notation ⌈x⌉ means the ceiling function, defined as the smallest integer that is not smaller than x).

  2. Pad M with copies of its last row until it has a power of two number of rows. As a result we obtain a matrix MRAP1​∈F2n×33.

Second round of RAP:

The verifier sends challenges α,z∈F (or the prover samples them from the transcript). Additional columns are added to incorporate the memory constraints. To define them the prover follows these steps:

  1. Stack the rows of the submatrix of M defined by the columns pc, dst_addr, op0_addr, op1_addr into a vector a of length 4(L+L′) (this means that the first entries of a are pc[0], dst_addr[0], op0_addr[0], op1_addr[0], pc[1], dst_addr[1],...).
  2. Stack the the rows of the submatrix defined by the columns inst, dst, op0, op1 into a vector v of length 4(L+L′).
  3. Define MMem​∈F(L+L′)×2 to be the matrix with columns a, v.
  4. Define MMemRepl​∈F(L+L′)×2 to be the matrix that's equal to MMem​ in the first L−Lpub​ rows, and its last Lpub​ entries are the addresses and values of the actual public memory.
  5. Sort MMemRepl​ by the first column in increasing order. The result is a matrix MMemReplSorted​ of size (L+L′)×2. Denote its columns by a' and b'.
  6. Compute the vector p of size 4(L+L′) with entries pi​=j=0∏i​z−(αai​+vi​)z−(αai′​+vi′​)​
  7. Reshape the matrix MMemReplSorted​ into a (L+L′)×8 in row-major. Reshape the vector p into a (L+L′)×4 matrix in row-major.
  8. Concatenate these 12 rows. The result is a matrix M′ of size (L+L′)×12.
  9. Pad M′ with copies of its last row until it has a power of two number of rows. As a result we obtain a matrix MRAP2​∈F2n×12.

Vector commitment scheme

Given a vector A=(y0​,…,yL​). The operation Commit(A) returns the root r of the Merkle tree that has the hash of the elements of A as leaves.

For i∈[0,2n+k), the operation open(A,i) returns yi​ and the authentication path s to the Merkle tree root.

The operation Verify(i,y,r,s) returns Accept or Reject depending on whether the i-th element of A is y. It checks whether the authentication path s is compatible with i, y and the Merkle tree root r.

Notation

In our cases the sets A will be of the form A=(f(a),f(ab),f(ab2),…,f(abL)) for some elements a,b∈F. It will be convenient to use the following abuse of notation. We will write Open(A,abi) to mean Open(A,i). Similarly, we will write Verify(abi,y,r,s) instead of Verify(i,y,r,s). Note that this is only notation and Verify(abi,y,r,s) is only checking that the y is the i-th element of the commited vector. It is not checking that it is an evaluation of a function at abi.

Protocol

Prover

Round 0: Transcript initialization

  • Start a transcript.
  • (Strong Fiat Shamir) Commit to the set of coefficientes of the transition and boundary polynomials, and add the commitments to the transcript.

Round 1: Build RAP

Round 1.1: Interpolate original trace
  • Obtain the trace of the program MRAP1​∈F2n×m′
  • For each column Mj​ of the matrix MRAP1​, interpolate its values at the domain DS​ and obtain polynomials tj​ such that tj​(gi)=Mi,j​.
  • Compute [tj​]:=Commit(tj​(DLED​)) for all j=1,…,m′.
  • Add [tj​] to the transcript in increasing order.
Round 1.2: Commit extended trace
  • Sample random values a1​,…,al​ in F from the transcript.
  • Use a1​,…,al​ to build MRAP2​∈F2n×m′′ following the specifications of the RAP process.
  • For each column M^j​ of the matrix MRAP2​, interpolate its values at the domain DS​ and obtain polynomials tm′+1​,…,tm′+m′′​ such that tj​(gi)=M^i,j​.
  • Compute [tj​]:=Commit(tj​(DLED​)) for all j=m′+1,…,m′+m′′.
  • Add [tj​] to the transcript in increasing order for all j=m′+1,…,m′+m′′.

Round 2: Compute composition polynomial

  • Sample α1B​,…,αmB​ and β1B​,…,βmB​ in F from the transcript.
  • Sample α1T​,…,αnT​T​ and β1T​,…,βnT​T​ in F from the transcript.
  • Compute Bj​:=ZjB​tj​−PjB​​.
  • Compute Ck​:=ZkT​PkT​(t1​,…,tm​,t1​(gX),…,tm​(gX))​.
  • Compute the composition polynomial H:=k∑​(αkT​XD−deg(Ck​)+βkT​)Ck​+j∑​(αjB​XD−deg(Bj​)+βjB​)Bj​
  • Decompose H as H=H1​(X2)+XH2​(X2)
  • Compute commitments [H1​] and [H2​].
  • Add [H1​] and [H2​] to the transcript.

Round 3: Evaluate polynomials in out of domain element

  • Sample from the transcript until obtaining z∈F∖DLDE​.
  • Compute H1​(z2), H2​(z2), and tj​(z) and tj​(gz) for all j.
  • Add H1​(z2), H2​(z2), and tj​(z) and tj​(gz) for all j to the transcript.

Round 4: Compute and run FRI on the Deep composition polynomial

  • Sample γ, γ′, and γ1​,…,γm​, γ1′​,…,γm′​ in F from the transcript.
  • Compute p0​ as γX−z2H1​−H1​(z2)​+γ′X−z2H2​−H2​(z2)​+j∑​γj​X−ztj​−tj​(z)​+γj′​X−gztj​−tj​(gz)​
Round 4.1.k: FRI commit phase
  • Let D0​:=DLDE​, and [p0​]:=Commit(p0​(D0​)).
  • Add [p0​] to the transcript.
  • For k=1,…,n do the following:
    • Sample ζk−1​ from the transcript.
    • Decompose pk−1​ into even and odd parts, that is, pk−1​=pk−1odd​(X2)+Xpk−1even​(X2).
    • Define pk​:=pk−1odd​(X)+ζk−1​pk−1even​(X).
    • If k<n:
      • Let L such that ∣Dk−1​∣=2L. Define Dk​:=(d12​,…,dL2​), where Dk−1​=(d1​,…,d2L​).
      • Let [pk​]:=Commit(pk​(Dk​)).
      • Add [pk​] to the transcript.
  • pn​ is a constant polynomial and therefore pn​∈F. Add pn​ to the transcript.
Round 4.2: FRI query phase
  • For s=0,…,Q−1 do the following:
    • Sample random index ιs​∈[0,2n+l] from the transcript and let υs​:=ωιs​.
    • Compute Open(p0​(D0​),υs​).
    • Compute Open(pk​(Dk​),−υs2k​) for all k=0,…,n−1.
Round 4.3: Open deep composition polynomial components
  • Compute Open(H1​(DLDE​),υ0​), Open(H2​(DLDE​),υ0​).
  • Compute Open(tj​(DLDE​),υ0​) for all j=1,…,m.

Build proof

  • Send the proof to the verifier: Π=()​{[tj​],tj​(z),tj​(gz):0≤j<m},[H1​],H1​(z2),[H2​],H2​(z2),{[pk​]:0≤k<n},pn​,{Open(p0​(D0​),υs​):0≤s<Q}),{Open(pk​(Dk​),−υs2k​):0≤k<n,0≤s<Q},Open(H1​(DLDE​),υ0​),Open(H2​(DLDE​),υ0​),{Open(tj​(DLDE​),υ0​):0≤j<m},​​

Verifier

Notation

  • Bold capital letters refer to commitments. For example H1​ is the claimed commitment [H1​].
  • Greek letters with superscripts refer to claimed function evaluations. For example τjz​ is the claimed evaluation tj​(z).
  • Gothic letters refer to authentication paths (e.g. H1​ is the authentication path of the opening of H1​).

Input

  • Add Tj​ to the transcript for j=m′+1,…,m′+m′′.
  • Sample α1B​,…,αmB​ and β1B​,…,βmB​ in F from the transcript.
  • Sample α1T​,…,αnT​T​ and β1T​,…,βnT​T​ in F from the transcript.
  • Add H1​ and H2​ to the transcript.
  • Sample z from the transcript.
  • Add η1z2​, η2z2​, τjz​ and τjgz​ to the transcript.

      • Solve the following system of equations on the variables G,H πkυs2k​​πk−υs2k​​​=G+υs2k​H=G−υs2k​H​
      • Define πk+1υs2k+1​​:=G+ζk​H

    • Verify((υ0​,η1υ0​​),H1​,h1​).
    • Verify((υ0​,η2υ0​​),H2​,h2​).
    • Verify((υ0​,τjυ0​​),Tj​,Tj​) for all 0≤j<m.
